# 사용자 여정 맵 (User Journey Map)

## UX-MVP-001: 게시글 CRUD 플로우

### 개요
이 문서는 커뮤니티 서비스의 핵심 기능인 게시글 CRUD(Create, Read, Update, Delete) 작업에 대한 사용자 여정을 정의합니다.

---

## 1. 게시글 작성 (Create) 플로우

### 여정 다이어그램

```
[메인 피드] ──▶ [작성 버튼 클릭] ──▶ [에디터 페이지] ──▶ [내용 입력] ──▶ [발행 버튼] ──▶ [게시 완료]
     │               │                    │               │               │              │
     ▼               ▼                    ▼               ▼               ▼              ▼
  탐색 중          기대감           집중/창작 모드      몰입            기대/긴장       만족/성취감
```

### 단계별 상세

| 단계 | 사용자 액션 | 시스템 응답 | 감정 포인트 |
|------|------------|------------|------------|
| 1. 진입 | 메인 피드에서 "글쓰기" 버튼 클릭 | 에디터 페이지로 전환 (0.3초 이내) | 기대감, 동기부여 |
| 2. 작성 | 제목 입력 | 실시간 글자 수 표시 | 집중, 창작 의욕 |
| 3. 본문 | 본문 내용 작성 | 자동 저장 (30초마다), 임시저장 알림 | 몰입, 안정감 |
| 4. 미리보기 | 미리보기 버튼 클릭 (선택) | 실제 게시될 형태 미리보기 | 확인, 점검 |
| 5. 발행 | "발행하기" 버튼 클릭 | 로딩 표시 → 성공 피드백 | 기대, 긴장 |
| 6. 완료 | 게시글 상세 페이지로 이동 | 게시 완료 토스트 메시지 | 만족, 성취감 |

### 예외 상황 처리

| 상황 | 시스템 응답 | 사용자 감정 대응 |
|------|------------|-----------------|
| 필수 항목 미입력 | 해당 필드 하이라이트 + 안내 메시지 | 혼란 방지, 명확한 가이드 |
| 네트워크 오류 | 자동 재시도 + 임시저장 안내 | 불안감 해소, 작성 내용 보존 |
| 발행 실패 | 오류 메시지 + 재시도 버튼 | 좌절감 최소화, 해결책 제시 |

## 2. 게시글 조회 (Read) 플로우

### 여정 다이어그램

```
[메인 피드] ──▶ [게시글 카드 클릭] ──▶ [상세 페이지 로딩] ──▶ [콘텐츠 소비] ──▶ [인터랙션]
     │                │                      │                    │               │
     ▼                ▼                      ▼                    ▼               ▼
  탐색/호기심       관심/선택             기대감              몰입/흥미         참여/공감
```

### 단계별 상세

| 단계 | 사용자 액션 | 시스템 응답 | 감정 포인트 |
|------|------------|------------|------------|
| 1. 탐색 | 피드 스크롤 | 무한 스크롤 로딩, 스켈레톤 UI | 탐색, 호기심 |
| 2. 선택 | 관심 게시글 클릭 | 페이지 전환 애니메이션 | 관심, 기대 |
| 3. 로딩 | 대기 | 스켈레톤 UI → 콘텐츠 표시 (1초 이내) | 기대감 |
| 4. 소비 | 콘텐츠 읽기 | 스크롤 진행률 표시 (선택적) | 몰입, 흥미 |
| 5. 반응 | 좋아요, 댓글, 북마크 | 즉각적인 피드백 애니메이션 | 참여, 공감 |

## 3. 게시글 수정 (Update) 플로우

### 여정 다이어그램

```
[게시글 상세] ──▶ [수정 버튼] ──▶ [에디터 로드] ──▶ [내용 수정] ──▶ [저장] ──▶ [완료]
      │               │               │               │             │          │
      ▼               ▼               ▼               ▼             ▼          ▼
   확인/검토       수정 의도        내용 확인       집중/수정     기대감     안도/만족
```

### 단계별 상세

| 단계 | 사용자 액션 | 시스템 응답 | 감정 포인트 |
|------|------------|------------|------------|
| 1. 진입 | 본인 게시글에서 "수정" 버튼 클릭 | 에디터 페이지로 전환 | 수정 의도 |
| 2. 로드 | 대기 | 기존 내용 자동 로드 | 내용 확인 |
| 3. 편집 | 제목/본문 수정 | 변경사항 추적, 자동 저장 | 집중, 개선 의지 |
| 4. 저장 | "저장하기" 버튼 클릭 | 로딩 표시 → 성공 피드백 | 기대감 |
| 5. 완료 | 수정된 게시글 확인 | 수정 완료 토스트 + 상세 페이지 이동 | 안도, 만족 |

### 권한 처리

| 상황 | 시스템 응답 |
|------|------------|
| 본인 게시글 | 수정/삭제 버튼 표시 |
| 타인 게시글 | 수정/삭제 버튼 미표시 |
| 비로그인 상태 | 로그인 유도 모달 |

## 4. 게시글 삭제 (Delete) 플로우

### 여정 다이어그램

```
[게시글 상세] ──▶ [삭제 버튼] ──▶ [확인 모달] ──▶ [삭제 확인] ──▶ [삭제 처리] ──▶ [피드 이동]
      │               │               │               │               │              │
      ▼               ▼               ▼               ▼               ▼              ▼
   결정/고민       삭제 의도       재확인/망설임    최종 결정       처리 중        정리/해소
```

### 단계별 상세

| 단계 | 사용자 액션 | 시스템 응답 | 감정 포인트 |
|------|------------|------------|------------|
| 1. 시작 | "삭제" 버튼 클릭 | 확인 모달 표시 | 삭제 의도 |
| 2. 확인 | 모달 내용 확인 | "정말 삭제하시겠습니까?" 메시지 | 재확인, 망설임 |
| 3. 결정 | "삭제" 또는 "취소" 클릭 | 사용자 선택에 따른 처리 | 최종 결정 |
| 4. 처리 | 대기 | 삭제 진행 로딩 | 처리 중 |
| 5. 완료 | 메인 피드로 이동 | 삭제 완료 토스트 메시지 | 정리, 해소 |

### 안전장치

- **확인 모달**: 실수로 인한 삭제 방지
- **명확한 문구**: "이 작업은 되돌릴 수 없습니다"
- **시각적 경고**: 삭제 버튼은 빨간색 계열로 구분

## 핵심 UX 원칙

1. **즉각적인 피드백**: 모든 액션에 0.3초 이내 시각적 반응
2. **안전망 제공**: 자동 저장, 확인 모달로 사용자 실수 방지
3. **명확한 상태 표시**: 현재 어떤 단계인지 항상 인지 가능
4. **긍정적 강화**: 작업 완료 시 성취감을 주는 피드백
5. **오류 복구**: 문제 발생 시 명확한 해결책 제시
